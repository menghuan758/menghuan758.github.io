小米路由器搭建git服务

1.前言

首先，我要声明，本次尝试失败了。没能在小米路由器上成功搭建对外开放的git服务。但希望我的尝试对你有所启发。本次并未尝试重刷第三方路由系统，使用小米路由系统开发版。具体ROM请访问：小米路由官网下载页

  • 设备：小米路由器第二代
  • 操作平台：OS X 系统

开始之前，请确认您已打开路由器SSH。具体方法请访问：小米路由器开发页
并且可以通过相关工具进行连接。
注：开启SSH过程中发生问题的话，请严格执行官方给出的步骤。

登陆SSH后，如果没有写入权限，可以执行以下命令，以读写权限重新挂载根目录。

失败原因：我这只能使用二级运营商宽带，鹏博士宽带，没有对外开发端口，无法实现内网至外网穿透，故以失败告终。
具体开放端口查询方式，请自行通过关键字搜索。
如若可实现外网访问，即可成功实现服务，具体方案请继续阅读。

2.目标

本次服务搭建的目标很简单，在路由器中搭建git仓库，开启外网穿透服务，实现远程提交代码至路由器。

3.实现方案

3.1 方案一

一般我们第一反应是使用gitlab搭建，功能强大。可惜这个方案不可行，因为gitlab要求配置较高，最小内存为1G，故无法搭建。

3.2 方案二

其实我们搭建git服务端一般仅为了个人使用，没必要搭建gitlab这种大型站点，使用git原生方法建立仓库即可。
具体参考方案：

  • 直接使用git命令搭建仓库
  • 在openwrt上安装git的教程

参考上面两篇博文，方法很显然，我们要在路由器上安装git。

操作步骤：
进入/etc/目录编辑opkg.conf文件（注意备份）
更改opkg镜像源：镜像源文件地址

baidu.openwrt.bobiji.com 百度节点
in.openwrt.bobiji.com in节点

注意：上述镜像源只能使用bcm53xx，不能使用brcm47xx等，软件是不兼容的，无法安装。

使用opkg命令安装git

使用参数的目的，主要是为了解决依赖问题，无法安装。加了参数即可安装，不用担心，亲测依赖装不装无所谓。
如果您真的想安装依赖，可以从这里下载依赖包，镜像源中无法找到这个处理器版本的依赖安装包，只能下载之后，传入路由器的/tmp/目录，使用opkg和ipk文件名安装。
如此，即可安装成功。
可惜安装完成之后git命令还是无法使用，加入环境变量也不可以，可能该软件依然不能完全兼容小米路由器固件。
具体原因，无法查明，各位可以完成以上步骤，分析原因。
如有余力更可分析源码，开发一份git安装包ipk兼容小米路由固件，供大家使用，在此感谢。

3.3 方案三

由于方案二尝试，路由器系统没能成功使用git，便想到将路由硬盘远程挂载到本地，使用电脑本机初始化git版本控制仓库。
这在路由器局域网内不难实现，mac系统下挂载的硬盘可以通过 /Volumes/路由器名 访问文件。切换到对应文件下初始化仓库服务端即可。

至于非局域网内如何将路由器硬盘挂载到本机电脑，我就没有再尝试了，如果各位同学可以远程登录路由器SSH倒是可以尝试挂载。

结尾

本次尝试虽然失败了，但探究过程还是值得分享的。
如果你也无法成功，可以像我一样尝试自行购买vps搭建服务。
推荐一个搬瓦工vps，国外主机还可以搭建vpn服务(并非广告，大家可以忽略)。
最低版本价格20刀左右，不过同样无法搭建gitlab。
希望本文对各位有所帮助。bye！
